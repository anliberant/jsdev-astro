When Node.js cannot locate the **mem‑fs** package, it usually means the dependency is missing, incorrectly installed, or incompatible with your project setup. Here are the most reliable ways to fix it.

## 1. Install the Package (Local Install Recommended)

```bash
npm install mem-fs --save
# or
yarn add mem-fs
```

Avoid global installs unless required:
```bash
npm install -g mem-fs   # not recommended for project use
```

## 2. Check `package.json`

```json
"dependencies": {
  "mem-fs": "^1.0.0"
}
```

If missing → reinstall.

## 3. Clear Cache & Reinstall Dependencies

```bash
npm cache clean --force
rm -rf node_modules
npm install
```

Yarn:

```bash
yarn cache clean
rm -rf node_modules
yarn install
```

## 4. Fix Import / Require Mismatch

If using **CommonJS**:

```js
const memFs = require('mem-fs');
```

If using **ESM** (package.json must include `"type": "module"`):

```js
import memFs from 'mem-fs';
```

❗ Mismatched module systems cause this error very often.

## 5. Verify Node.js Compatibility

```bash
node -v
npm -v
```

Older Node versions may not support some package builds.

## 6. Install a Compatible Version

```bash
npm install mem-fs@1.1.3
```

Check versions here:  
https://www.npmjs.com/package/mem-fs

## 7. Local vs Global Conflict

If you installed mem‑fs globally but not locally:

```bash
npm link mem-fs
```

Better: always install locally.

## 8. Using mem‑fs with Yeoman? Use This Template

```js
const memFs = require('mem-fs');
const editor = require('mem-fs-editor');

const store = memFs.create();
const fs = editor.create(store);

// fs.write(), fs.commit(), etc.
```

## Summary Table

| Step | Fix |
|------|-----|

## Conclusion

The “Cannot find package 'mem‑fs'” error is almost always caused by a missing install, module format mismatch, or corrupted node_modules. Use the steps above to quickly fix the issue and get your project running smoothly.
